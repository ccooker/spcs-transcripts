## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/students/:id/academics | Untrusted: studentId and body from browser; staff role required |
| /api route handler → service layer | Server-internal: studentId validated by parseStudentId; recordId trust deferred to IDOR check in service |
| service → Prisma | Internal: parameterized queries; no string concatenation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering / Info Disclosure | academicResult service update/delete | mitigate | IDOR guard: `if (!existing \|\| existing.studentId !== studentId) throw AcademicResultNotFoundError` — always 404, never 403 (ASVS V4.2) |
| T-03-02 | Tampering | academicResult + activity POST/PATCH routes | mitigate | Zod `.strict()` on createAcademicResultSchema and createActivitySchema — unknown fields rejected with 400 |
| T-03-03 | Repudiation | academicResult + activity services (create/update/delete) | mitigate | `logAudit()` called inside service functions after every write; not in route handlers |
| T-03-04 | Spoofing | all /academics and /activities routes | accept | `validateJwt` + `resolveUser` global middleware from Phase 1 inherited by all routes; no per-router setup needed |
| T-03-SC | Tampering | npm/shadcn installs | mitigate | No new npm packages in this plan; @tanstack/react-query installed in Plan 03-01b (verified [OK] in 03-RESEARCH.md) |
