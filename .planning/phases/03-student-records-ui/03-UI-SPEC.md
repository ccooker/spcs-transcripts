## Interaction Specifications

### Record Add/Edit Dialog — Shared Pattern

All four list-section dialogs (academics, activities, awards, work experience) follow this pattern:

- Trigger: "Add [type]" button in card header OR Pencil icon in table row
- Dialog opens at center (`DialogContent`); form fields inside
- Form uses React Hook Form + Zod; inline `FormMessage` for validation errors
- Submit: POST (add) or PATCH (edit) → close dialog → sonner toast → refetch section data
- Cancel: close dialog, discard unsaved changes

### Record Delete — Shared Pattern

All four list sections share `RecordDeleteDialog` (AlertDialog):

- Trigger: Trash icon button in table row
- `AlertDialogTitle`: "Delete this entry?"
- `AlertDialogDescription`: "This action cannot be undone." (no data is soft-deleted — hard delete for record entries)
- `AlertDialogCancel`: "Keep entry"
- `AlertDialogAction variant="destructive"`: "Delete"
- On confirm: DELETE API → close dialog → sonner toast "Entry deleted" → refetch section data

### Month/Year Picker — Shared Component

Used in Activities, Awards (single date), Work Experience dialogs.

- Two side-by-side `Select` components inside a `flex gap-2` wrapper
- Month Select: 12 options ("Jan" → "Dec") using full month name in option; short name in trigger (space-constrained)
- Year Select: current year down to 10 years ago (rolling); ascending order in list
- End date: additional `Checkbox` "Ongoing / Present" — when checked, disables the month/year selects and submits null to API

### Career Goals Pre-population

When opening "Update career goals" dialog with an existing version:
- Pre-check all previously selected interest checkboxes
- Pre-fill description textarea with previous description
- Staff sees current state as starting point; saving always creates a new version

### Section Loading State

Each `RecordSectionCard` shows skeleton rows independently — sections load in parallel, not sequentially:

| Section | Skeleton pattern |
|---------|-----------------|
| Academics, Activities, Awards, Work Exp | 3 skeleton rows matching table column layout |
| Career Goals | Skeleton block for interests row + skeleton lines for description |
| Notes | Skeleton attribution line + 2 skeleton content lines × 2 notes |

### Error States

Each section card independently handles API errors:

- API failure fetching section data: show `Alert variant="destructive"` inside card body with "Couldn't load [section]. Try again." and `Button variant="outline"` "Retry"
- Mutation failure (add/edit/delete): sonner toast error `"Couldn't save changes. Please try again."` — dialog stays open (does not close on failure)

| D-23 | After delete: toast + refresh | Sonner "Entry deleted" + refetch section query |

## Accessibility Baseline

Inherit WCAG 2.1 AA from 01-UI-SPEC.md and Phase 2 additions. Phase 3 additions:

| Requirement | Implementation |
|-------------|---------------|
| Record section headings | `<h2>` for each section title inside `RecordSectionCard` |
| Dialog focus trap | Radix Dialog default; first field receives focus on open |
| Delete dialog keyboard | AlertDialog: Enter triggers focused action; Escape closes |
| Checkbox group | `<fieldset>` + `<legend>` wrapping `CareerInterestsChecklist` |
| Textarea character count | `aria-describedby` linking textarea to `{n}/500` counter span |
| Append-only note textarea | `<label>` "Add a note" associated via `htmlFor` |
| Section loading | `aria-busy="true"` on each section card during data fetch |
| Toast announcements | Sonner `role="status"` default (inherited) |
| Month/year selects | Each `Select` has visible `Label`; both labeled as "Month" / "Year" within picker wrapper with `<legend>` |
| Read-only version history | Historical version entries use `<button>` expand toggle with `aria-expanded`; content shown/hidden via conditional render |
| Notes list | `<ul>` with `<li>` per note; attribution in `<time>` element with `dateTime` attribute |
