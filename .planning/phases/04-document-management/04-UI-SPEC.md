### Documents Table

**Columns:**

| Column | Field | Component | Notes |
|--------|-------|-----------|-------|
| File name | `originalFilename` | `text-sm font-semibold` truncated | `max-w-[240px] truncate`; full filename in `Tooltip` on hover |
| Type | `typeTag` | `Badge variant="secondary"` | Display label from Document Type Tag table above |
| Uploaded | `createdAt` | `text-sm text-muted-foreground` | Format: "12 Jun 2026" (`d MMM yyyy`) |
| Uploader | `uploader.displayName` | `text-sm text-muted-foreground` | Staff display name |
| Actions | — | `flex items-center gap-1` | Download + Delete icon buttons |

**Default sort:** `createdAt` descending — most recently uploaded document first.

**Pagination:** None in Phase 4. Render full list. Students are unlikely to accumulate more than a few dozen documents; the page scrolls naturally.

**Action buttons (per row):**

| Button | Icon | Variant | Size | Behaviour |
|--------|------|---------|------|-----------|
| Download | Lucide `Download` | `ghost` | `icon` | `aria-label="Download {originalFilename}"`; triggers `GET /api/students/:id/documents/:docId/download`; opens `Content-Disposition: attachment` response in same tab |
| Delete | Lucide `Trash2` | `ghost` | `icon` | `aria-label="Delete document"`; `className="hover:text-destructive"`; opens delete AlertDialog |

Both action buttons use `Button variant="ghost" size="icon"` — consistent with Phase 3 record table action column pattern.

### Upload Dialog

**Trigger:** "Upload document" button in `DocumentsSection` card header.

**Layout:**

```
┌─ Upload document ─────────────────────────────────────────────────────┐
│                                                                        │
│  PDF file *                                                            │
│  [ Choose file ]   report_card_2025.pdf ✓                             │
│  PDF only, max 25 MB                                                   │
│                                                                        │
│  Document type *                                                       │
│  [ Report Card ▾ ]                                                     │
│                                                                        │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  64%        [uploading]      │
│  (progress bar — hidden until upload starts)                           │
│                                                                        │
│                [ Discard upload ]  [ Upload document ]                │
└────────────────────────────────────────────────────────────────────────┘
```

**Dialog fields:**

| Field | Required | Component | Notes |
|-------|----------|-----------|-------|
| PDF file | Yes | Native `<input type="file" accept=".pdf">` wrapped in `<Button variant="outline">Choose file</Button>` | Button-only picker (D-17); native file dialog on click; selected filename displayed next to button after selection |
| Document type | Yes | `Select` | Options: Report Card, Certificate, Award Letter, Work Experience Letter, Reference Letter, Other (D-20); placeholder "Select type…" |

**Selected file display:** After file selection, show filename in `text-sm text-muted-foreground` next to "Choose file" button — e.g. `report_card_2025.pdf`. Truncate at 40 chars with ellipsis if needed.

**Progress bar behaviour (D-19):**

- Hidden (`hidden`) until upload starts
- Visible during upload: `Progress value={uploadPercent}` (0–100) with `w-full` and `mt-4`
- Progress driven by `XMLHttpRequest` `upload.onprogress` events
- Accompanied by `text-sm text-muted-foreground` label: "Uploading… {n}%" at right

**Dialog states:**

| State | Controls | Submit label |
|-------|----------|-------------|
| Idle (no file selected) | All enabled; Submit disabled | "Upload document" (disabled) |
| File selected, type selected | All enabled; Submit enabled | "Upload document" |
| Uploading | File input + type select + Discard upload + Submit all disabled | "Uploading…" |
| Upload error | Controls re-enabled; error toast displayed; progress bar hidden | "Upload document" |
| Upload success | Dialog closes; progress bar hidden | — |

**Submit flow:** `fetch` with `XMLHttpRequest` wrapping (for progress events) → `POST /api/students/:id/documents` with `Content-Type: multipart/form-data` → on success: close dialog, `toast.success("Document uploaded")`, refetch document list.

**Cancel button:** `Button variant="outline"` label "Discard upload". Disabled while uploading.

**File validation (client-side, before submit):**

| Check | Error Display |
|-------|--------------|
| File type not `.pdf` | Inline error below file input: "Only PDF files are accepted." |
| File size > 25 MB | Inline error below file input: "File exceeds the 25 MB limit. Choose a smaller file and try again." |

Server-side validation mirrors these checks; errors surfaced as toast if client-side check is bypassed.

### Delete Document — AlertDialog

**Trigger:** Trash2 icon button in document table row.

**Layout:**

```
┌─ Delete document ──────────────────────────────────────────────────────┐
│                                                                         │
│  This document will be removed from the list. The file is retained     │
│  in the system with an audit trail entry.                               │
│                                                                         │
│                       [ Keep document ]  [ Delete document ]           │
└─────────────────────────────────────────────────────────────────────────┘
```

**Controls:**

| Element | Component | Label |
|---------|-----------|-------|
| Cancel | `AlertDialogCancel` | "Keep document" |
| Confirm | `AlertDialogAction variant="destructive"` | "Delete document" |

**On confirm:** `DELETE /api/students/:id/documents/:docId` → `toast.success("Document deleted")` → refetch document list.

## Accessibility Baseline

Inherit WCAG 2.1 AA from 01-UI-SPEC.md, Phase 2, and Phase 3 additions. Phase 4 additions:

| Requirement | Implementation |
|-------------|---------------|
| Document section heading | `<h2>` for "Documents" inside `RecordSectionCard` (inherited pattern) |
| File input labeling | `<label htmlFor="doc-file-input">PDF file</label>` + `aria-required="true"` on native input |
| Type select labeling | `<Label htmlFor="doc-type-select">Document type</Label>` |
| Upload progress bar | `<Progress aria-label="Upload progress" aria-valuenow={uploadPercent} aria-valuemin={0} aria-valuemax={100} />` |
| Uploading state announcement | `aria-live="polite"` region wrapping progress bar + "Uploading… {n}%" text |
| Submit disabled state | `disabled` attribute on Upload button when preconditions unmet; `aria-disabled` for screen readers |
| Download button | `aria-label="Download {originalFilename}"` (unique per row) |
| Delete button | `aria-label="Delete document"` |
| Delete dialog | Focus trap via Radix AlertDialog default; Escape closes; Enter triggers focused action |
| File name truncation | `Tooltip` on truncated cell with full filename accessible via keyboard |
| Section loading | `aria-busy="true"` on section card during fetch (inherited `RecordSectionCard` pattern) |
| Toast announcements | Sonner `role="status"` default (inherited) |
