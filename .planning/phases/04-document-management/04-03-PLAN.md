**As a** careers staff member, **I want to** upload, organise, and retrieve PDF documents attached to a student's record, **so that** supporting evidence is stored securely alongside the student's data and accessible for transcript preparation.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → XHR POST multipart | File + typeTag from user input; client-side validation is advisory; server is authoritative |
| Browser → fetch GET download | JWT Bearer token must be included; download fetched via authenticated fetch, not window.open |
| React JSX → displayed filename | originalFilename from API response displayed in DOM; React auto-escapes; no dangerouslySetInnerHTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Information Disclosure | XHR Authorization header in browser memory | accept | Token is a short-lived MSAL access token acquired via acquireTokenSilent; stored in sessionStorage (Phase 1 decision); XHR sets it per-request in memory only |
| T-04-08 | Tampering | Client-side file type/size validation | accept | Client-side check is defence-in-depth UX only; server enforces MIME filter + magic bytes + 25 MB limit; bypass of client check still fails at server (T-04-01/T-04-02 from plan 04-01) |
| T-04-09 | Information Disclosure | XSS via originalFilename in table cell | mitigate | React renders text via JSX which auto-escapes HTML entities; no innerHTML or dangerouslySetInnerHTML used; filename is rendered as {doc.originalFilename} text node only |
| T-04-10 | Elevation of Privilege | Download button for another student's document | mitigate | Download URL includes studentId from current page's student prop; server IDOR guard rejects mismatched studentId (T-04-03 from plan 04-01); client cannot forge a different studentId without being on a different student's page |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | shadcn progress is installed via `npx shadcn@latest add progress` which is the official shadcn CLI; no new npm package added to package.json — shadcn copies source files locally |
</threat_model>
