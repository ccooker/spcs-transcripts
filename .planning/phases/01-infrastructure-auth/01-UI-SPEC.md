# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Infrastructure & Auth.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** Phase 1 UI is minimal — a login page, a post-login home page, and
> role-gated navigation. These tokens and components are the canonical foundation
> for all downstream phases (2–5). Every design decision made here carries forward.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | shadcn/ui (latest) | User input |
| Style | default (not new-york) | Default — cleaner for data-heavy internal tools |
| Preset | zinc base, blue-600 accent, Inter | User input + sensible default |
| Component library | Radix UI (via shadcn) | shadcn dependency |
| Icon library | Lucide React | User input |
| Font | Inter (self-hosted via fontsource) | User input |

### shadcn Initialization

> **For executor:** The `client/` directory does not exist yet. Run the following
> after `npm create vite@latest client -- --template react-ts`:

```bash
cd client
npm install
npx shadcn@latest init
```

When prompted, select:
- Style: **Default**
- Base color: **Zinc**
- CSS variables: **Yes**

Then install required Phase 1 components:

```bash
npx shadcn@latest add button card badge separator avatar dropdown-menu alert
```

Install Inter font:

```bash
npm install @fontsource-variable/inter
```

Import in `client/src/main.tsx`:

```ts
import '@fontsource-variable/inter';
```

Confirm `components.json` exists before proceeding to implementation.

### Font Application

Set Inter as the CSS variable font in `client/src/index.css` (inside `@layer base`):

```css
:root {
  --font-sans: 'Inter Variable', system-ui, sans-serif;
}
body {
  font-family: var(--font-sans);
}
```

## Spacing Scale

Declared values — all multiples of 4. Applied via Tailwind spacing utilities.

| Token | Value | Tailwind Class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, inline label padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, badge padding |
| md | 16px | `p-4` / `gap-4` | Default element spacing, form field gaps |
| lg | 24px | `p-6` / `gap-6` | Card internal padding, section spacing |
| xl | 32px | `p-8` / `gap-8` | Layout gaps, page section breaks |
| 2xl | 48px | `p-12` / `gap-12` | Major section breaks |
| 3xl | 64px | `p-16` / `gap-16` | Page-level vertical rhythm |

Exceptions:
- Microsoft Sign-in button: minimum height **44px** (WCAG 2.5.5 touch target; use `min-h-[44px]`)
- Login card max-width: **400px** centered (`max-w-sm` or `max-w-[400px]`)

## Typography

Exactly 4 sizes, 2 weights.

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Display | 28px | Semibold (600) | 1.2 | `text-2xl font-semibold leading-tight` | Page titles ("Welcome, {name}") |
| Heading | 20px | Semibold (600) | 1.2 | `text-xl font-semibold leading-tight` | Section headings, card titles |
| Body | 16px | Regular (400) | 1.5 | `text-base font-normal leading-relaxed` | Body copy, descriptions, nav labels |
| Label | 14px | Regular (400) | 1.4 | `text-sm font-normal leading-snug` | Form labels, metadata, role badge text |

**Weights used:** 400 (regular) and 600 (semibold) only. No 500, no 700.

## Color

Sourced from shadcn CSS variables (zinc base + blue-600 accent). Use CSS variable names — never hardcode hex in component code.

| Role | CSS Variable | Hex (reference) | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` / `--card` | `#FFFFFF` / `#FAFAFA` | Page background, card surfaces |
| Secondary (30%) | `--muted` / `--secondary` | `#F4F4F5` (zinc-100) | Sidebar bg, nav bg, table header rows |
| Accent (10%) | `--primary` | `#2563EB` (blue-600) | Reserved list below |
| Destructive | `--destructive` | `#DC2626` (red-600) | Destructive actions only |
| Foreground | `--foreground` | `#09090B` (zinc-950) | Body text |
| Muted foreground | `--muted-foreground` | `#71717A` (zinc-500) | Helper text, timestamps, secondary labels |
| Border | `--border` | `#E4E4E7` (zinc-200) | Card borders, dividers, input borders |

**Accent (`--primary` / `#2563EB`) reserved for:**
1. "Sign in with Microsoft" button background
2. Active navigation item indicator (left border or background highlight)
3. Primary action buttons in downstream phases
4. Focus ring on interactive elements (`ring-primary`)

Accent is **NOT** used for: decorative elements, background fills larger than a button, hover states on secondary elements.

**Dark mode:** Not in scope for v1. All CSS variables target light mode only.

## Screen Inventory (Phase 1)

Exactly 3 screens required by Phase 1 success criteria.

### Screen 1 — Login Page

**Route:** `/` (unauthenticated state)

**Layout:** Full-height centered column. Logo placeholder above card. Card contains CTA.

```
┌─────────────────────────────────────────┐
│          [School Logo Placeholder]      │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  SPCS Student Transcript System │    │
│  │  Sign in with your school       │    │
│  │  Microsoft account to continue. │    │
│  │                                 │    │
│  │  [ 🪟 Sign in with Microsoft ]  │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

**States:**
- Default: CTA button enabled
- Loading (redirect in progress): CTA button disabled, spinner inside button
- Error: Alert component below card with error copy (see Copywriting)

**Microsoft button styling:**
- Background: `--primary` (#2563EB) — accent colour
- Icon: Microsoft SVG logo (inline, not from Lucide) embedded as an `<img>` or inline SVG
- Min height: 44px (accessibility exception from Spacing Scale)
- Full width within card

### Screen 2 — Home / Dashboard Page (authenticated)

**Route:** `/` (authenticated state) or `/home`

**Layout:** Top navigation bar + content area.

```
┌──────────────────────────────────────────────────────────┐
│  SPCS Transcripts          [nav items - role gated]  [👤▾]│
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Welcome, {displayName}          [Staff] or [Admin]      │
│                                                          │
│  (Phase 2+ will populate this area with student search)  │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**Navigation items visible to Staff:** Students (placeholder, enabled in Phase 2)

**Navigation items visible to Admin only:** Settings (user management, branding config — enabled in later phases)

**User menu (avatar dropdown):**
- Display name
- Role badge
- "Sign out" menu item

**States:**
- Loading user (`/api/auth/me` in-flight): skeleton placeholder for welcome text and nav
- Loaded: full layout

### Screen 3 — Unauthorized Page (403)

**Route:** `/unauthorized`

**Layout:** Centered message, no nav sidebar links to admin areas.

```
┌─────────────────────────────────────────┐
│  You don't have permission              │
│  to access this page.                   │
│                                         │
│  [ ← Go to home ]                       │
└─────────────────────────────────────────┘
```

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Login page app title | "SPCS Student Transcript System" | Use full name on login; abbreviated in nav |
| Login page subtitle | "Sign in with your school Microsoft account to continue." | Source: user decision — Entra ID SSO only |
| Primary CTA | "Sign in with Microsoft" | Standard MS branding; do not alter wording |
| CTA loading state | "Signing in…" | Shown while MSAL redirect initiates |
| Home welcome heading | "Welcome, {displayName}" | `displayName` from `GET /api/auth/me` response |
| Role badge — Admin | "Admin" | Rendered as `<Badge variant="default">` |
| Role badge — Staff | "Staff" | Rendered as `<Badge variant="secondary">` |
| Nav app name (abbreviated) | "SPCS Transcripts" | Use in top nav header |
| Sign out label | "Sign out" | In user dropdown menu |
| Auth error heading | "Sign-in failed" | Source: user input |
| Auth error body | "Sign-in failed. Please try again." | Shown in `<Alert variant="destructive">` below login card |
| Session expired heading | "Session expired" | When MSAL token is invalidated |
| Session expired body | "Your session has expired. Please sign in again." | Shown on any 401 response from API |
| 403 page heading | "You don't have permission to access this page." | Role guard failure |
| 403 page CTA | "Go to home" | Button/link back to `/` |
| Loading skeleton label | (no text — visual only) | Skeleton shimmer, no copy needed |

**Destructive actions in Phase 1:** None. No delete/destroy operations appear in Phase 1 UI. Destructive confirmation patterns are deferred to Phase 2+.

## Accessibility Baseline

All Phase 1 components must meet WCAG 2.1 Level AA:

| Requirement | Implementation |
|-------------|---------------|
| Color contrast body | `#09090B` on `#FFFFFF` — 19.6:1 (exceeds 4.5:1) |
| Color contrast muted | `#71717A` on `#FFFFFF` — 4.6:1 (passes 4.5:1) |
| Color contrast accent (white on blue) | `#FFFFFF` on `#2563EB` — 4.7:1 (passes 4.5:1) |
| Focus visible | shadcn default `focus-visible:ring-2 ring-primary` on all interactive elements |
| Touch target | Sign-in button min-height 44px (WCAG 2.5.5) |
| Screen reader | Microsoft SVG icon in CTA has `aria-hidden="true"`; button has visible text label |
| Keyboard navigation | Full tab order through login card → CTA → no traps |
