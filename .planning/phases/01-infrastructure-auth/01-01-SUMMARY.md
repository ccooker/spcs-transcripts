## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Prisma 7 breaking change: datasource `url` property removed from schema**
- **Found during:** Task 1 (prisma validate)
- **Issue:** Prisma 7.8.0 no longer accepts `url = env("DATABASE_URL")` in the datasource block; validation exited with error P1012
- **Fix:** Removed `url` from datasource; changed generator to `provider = "prisma-client"` with `output = "../src/generated/prisma"`; created `prisma.config.ts` with `defineConfig({datasource:{url:env('DATABASE_URL')}})` from `prisma/config`
- **Files modified:** prisma/schema.prisma, prisma.config.ts (new)
- **Verification:** `prisma validate` exits 0
- **Committed in:** `8156313`

**2. [Rule 1 - Bug] Prisma 7 requires driver adapter — PrismaClient no longer reads DATABASE_URL directly**
- **Found during:** Task 1 (prisma client docs for v7)
- **Issue:** Prisma 7 requires `PrismaClient({ adapter })` with `@prisma/adapter-pg`; `new PrismaClient()` without adapter throws at runtime
- **Fix:** Installed `@prisma/adapter-pg` and `pg`; updated `src/lib/prisma.ts` to create `PrismaPg({ connectionString: process.env.DATABASE_URL! })` and pass to PrismaClient
- **Files modified:** package.json, src/lib/prisma.ts
- **Verification:** `prisma generate` succeeds; `tsc --noEmit` exits 0
- **Committed in:** `8156313`

**3. [Rule 3 - Blocking] No PostgreSQL or Docker available — prisma migrate dev cannot run**
- **Found during:** Task 1 (environment check)
- **Issue:** `psql` and `docker` not found in PATH; `prisma migrate dev --name init` requires a live database
- **Fix:** Used `npx prisma migrate diff --from-empty --to-schema prisma/schema.prisma --script` to generate the SQL without a DB connection; created migration directory and SQL file manually
- **Files modified:** prisma/migrations/20260611045200_init/migration.sql (new)
- **Verification:** `ls server/prisma/migrations/` shows the migration directory with migration.sql
- **Committed in:** `8156313`

**4. [Expected - RED Phase] TypeScript compile errors in test files**
- **Found during:** Task 2 (tsc --noEmit after test suite creation)
- **Issue:** Test files intentionally import non-existent modules (`middleware/requireRole.js`, `services/audit.js`) — plan explicitly stated "TypeScript import errors are acceptable here"
- **Fix:** No fix needed — this is the intended RED state. Task 1 verified `tsc --noEmit` exits 0 on non-test code
- **Impact:** Test files will remain RED until Plan 03 implements the middleware and services

- Then run: `cd server && npx prisma migrate deploy` to apply the migration SQL
