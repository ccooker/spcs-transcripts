## Color

**Inherited CSS variables from `client/src/index.css` / 01-UI-SPEC.md. Never hardcode hex in components.**

| Role | CSS Variable | Usage in Phase 2 |
|------|-------------|------------------|
| Dominant (60%) | `--background` | Page background, table body rows |
| Secondary (30%) | `--muted` | Table header row (`bg-muted/50`), cohort group header rows (`bg-muted/50`), filter toolbar background optional |
| Accent (10%) | `--primary` | See reserved list below |
| Destructive | `--destructive` | Archive button, archive dialog confirm, validation errors |
| Muted foreground | `--muted-foreground` | Empty states, pagination info, cohort summary counts |
| Border | `--border` | Table borders, form field borders, card outlines |

**Accent (`--primary`) reserved for (Phase 2):**

1. "Add student" primary button on list page
2. "Save" / "Create student" submit buttons on forms
3. Active **Students** nav link indicator (bottom border or text color — match Phase 1 nav active pattern)
4. Finalised transcript status badge (`Badge variant="default"`)
5. Focus rings on all interactive elements

**Accent is NOT used for:** table row hover, cohort header backgrounds, Draft/None status badges, pagination inactive state.

### Transcript Status Badge Variants

| Status | Badge Variant | Label Copy |
|--------|--------------|------------|
| `NONE` | `secondary` | "None" |
| `DRAFT` | `outline` | "Draft" |
| `FINALISED` | `default` | "Finalised" |

All students show **None** until Phase 5 enables status changes. Filter UI must still expose all three values (D-08).

### Archived Row Visual Treatment

When admin views archived students (`includeArchived=true`):

- Row background: `bg-muted/30`
- Append muted label after name: `(Archived)` in `text-muted-foreground text-sm`
- Restore action uses `Button variant="outline"` — not destructive

## Visual Focal Points

Each primary screen has one dominant element that draws the eye first. Secondary chrome (nav, toolbars, pagination) supports but does not compete.

| Screen | Focal point | Implementation |
|--------|-------------|----------------|
| Student list (`/students`) | **Data table** | Full-width table occupies ~60% of viewport below toolbar; page title and "Add student" are secondary; cohort group headers guide scan within the table |
| Create student (`/students/new`) | **Profile form card** | Single Card with form fields centered in content area; page title "Add student" above; action row below card |
| Student detail (`/students/:id`) | **Student name header** | `text-2xl font-semibold` full name + subtitle line is the hero block; profile card and records placeholder are supporting content below |

## Screen Inventory

### Screen 1 — Student List

**Route:** `/students`  
**Requirements:** STU-02 (list), NAV-01, NAV-02, NAV-03, D-09, D-10, D-11  
**Focal point:** Data table (see Visual Focal Points)

**Layout:**

```
┌──────────────────────────────────────────────────────────────────────────┐
│  [AppShell header — Students nav active]                                 │
├──────────────────────────────────────────────────────────────────────────┤
│  Students                                           [ Add student ]      │
│                                                                          │
│  ┌─────────────────────────────────────────────────────────────────────┐ │
│  │ [Search by name___________] [Search students]  Form [All ▾]  Status [All ▾] │ │
│  │ [Admin only: ☐ Show archived]                                        │ │
│  └─────────────────────────────────────────────────────────────────────┘ │
│                                                                          │
│  ┌─────────────────────────────────────────────────────────────────────┐ │
│  │ ▼ Form 1          Draft: 0 · Finalised: 0 · None: 42              │ │
│  │   Chan Tai Man    Form 1   Class of 2029   S2024001   [None]   →    │ │
│  │   ...                                                                │ │
│  │ ▼ Form 4          Draft: 0 · Finalised: 0 · None: 38              │ │
│  │   ...                                                                │ │
│  └─────────────────────────────────────────────────────────────────────┘ │
│                                                                          │
│  Showing 1–50 of 312 students          [ ← Prev ]  Page 1 of 7  [Next → ] │
└──────────────────────────────────────────────────────────────────────────┘
```

**Page header row:**

- Left: `<h1>` "Students" (`text-2xl font-semibold`)
- Right: `<Button>` "Add student" → navigates to `/students/new` (primary/accent)

**Search & filter toolbar** (`flex flex-wrap items-end gap-4 mb-6`):

| Control | Component | Behaviour |
|---------|-----------|-----------|
| Name search input | `Input` | Placeholder: "Search by name"; controlled local state; **does not** trigger API on change |
| Search button | `Button variant="default"` | Label: "Search students"; applies `q` param and refetches (D-11) |
| Enter key in search input | — | Same as Search students button click |
| Form level filter | `Select` | Options: "All forms", "Form 1" … "Form 6" (D-05, D-06); changing value immediately refetches |
| Transcript status filter | `Select` | Options: "All statuses", "Draft", "Finalised", "None" (D-08); immediate refetch |
| Show archived toggle | `Checkbox` or switch-style toggle | **Admin only** (D-14); sets `includeArchived=true`; hidden for Staff |

**Data table columns** (sortable via column header click — D-09):

| Column | Field | Sortable | Notes |
|--------|-------|----------|-------|
| Full name | `fullName` | Yes | Primary identifier; row click navigates to `/students/:id` |
| Form | `formLevel` | Yes | Display "Form 1" … "Form 6" |
| Graduation year | `graduationYear` | Yes | Display "Class of {year}" (D-07) |
| Student ID | `schoolStudentId` | Yes | |
| Transcript status | `transcriptStatus` | Yes | Badge per status variant table above |
| Actions | — | No | Chevron-right icon or "View" link to detail |

**Default sort:** `formLevel` ascending, then `fullName` ascending — supports cohort grouping (D-10, RESEARCH discretion).

**Cohort group headers (NAV-03, D-10):**

- Inject `<CohortGroupHeader>` row when `formLevel` changes between consecutive data rows
- Full-width cell (`colSpan={columns.length}`), `bg-muted/50`, non-hoverable
- Content: **"{Form N}"** (semibold) + summary **"Draft: {n} · Finalised: {n} · None: {n}"**
- Counts computed from **current filtered/search result set on screen** (RESEARCH A1 — not global DB totals)
- When sort order is not form-based, suppress group headers (no misleading groups)

**Pagination:**

- Server-side, `pageSize: 50` (RESEARCH recommendation)
- Footer: "Showing {start}–{end} of {total} students" (`text-sm text-muted-foreground`)
- Prev/Next buttons; disabled at boundaries
- TanStack `manualPagination: true`

**Row interaction:**

- Click row (or name cell) → `/students/:id`
- Cursor:pointer on clickable rows
- Admin archived rows: show Restore button in actions column instead of chevron

**States:**

| State | Visual |
|-------|--------|
| Loading (initial / refetch) | 5 skeleton rows matching table column layout; toolbar controls remain visible but disabled |
| Empty — no students yet | See Copywriting: empty list |
| Empty — no search results | See Copywriting: no results |
| Error — API failure | `Alert variant="destructive"` above table + "Try again" button |
| Loaded | Full table |

| Student phone | No | `Input` | |

### Screen 3 — Student Detail / Edit

**Route:** `/students/:id`  
**Requirements:** STU-02, D-04, D-13, D-14, D-15  
**Focal point:** Student name header (see Visual Focal Points)

**Layout:**

```
┌──────────────────────────────────────────────────────────────────────────┐
│  [AppShell]                                                              │
├──────────────────────────────────────────────────────────────────────────┤
│  ← Back to students                                                      │

│                                                                          │
│  ┌─ Profile ─────────────── [ Edit ] [ Discard changes ] [ Save changes ]─┐│
│  │  (read-only view OR inline edit fields — same as create form)        ││
│  └──────────────────────────────────────────────────────────────────────┘│
│                                                                          │
│  ┌─ Student records ──────────────────────────────────────────────────┐│
│  │  Academic results, activities, awards, and other records will be   ││
│  │  available here in a future update.                                 ││
│  └──────────────────────────────────────────────────────────────────────┘│
│                                                                          │
│  [Admin + archived: Restore student button]                              │
└──────────────────────────────────────────────────────────────────────────┘
```

**Header:**

- Display name: `text-2xl font-semibold`
- Subtitle line: `{Form label} · Class of {year} · {schoolStudentId}` — `text-sm text-muted-foreground`
- Transcript status badge (right of name)
- **Archive** button: `Button variant="destructive"` — visible for active (non-archived) students; Staff + Admin (D-14)

**Profile section:**

- Default: read-only field display (label + value pairs, `gap-4` grid, 2 columns on `md+`)
- **Edit** toggles to same fields as create form; **Save changes** submits PATCH
- **Discard changes** (`Button variant="outline"`) reverts to read-only without navigation — visible only in edit mode

**Phase 3 placeholder (D-04):**

- Card titled "Student records"
- Body: "Academic results, activities, awards, and other records will be available here in a future update."
- Muted card styling (`text-muted-foreground`); no interactive elements

**Admin restore (D-16):**

- When viewing an archived student (admin only): show `Button variant="outline"` "Restore student" in page header area
- Staff never see restore UI or archived students in default list

**Archive flow (D-15):**

- Click "Archive" → open `ArchiveStudentDialog` (AlertDialog)
- Dialog content:
  - Title: "Archive student"
  - Description: explains student hidden from list, data retained
  - Label: "Type **{fullName}** to confirm"
  - Text input (controlled)
  - Dismiss: `AlertDialogCancel` — label **"Keep student"**
  - Confirm: `AlertDialogAction variant="destructive"` — label **"Archive student"**; **disabled** until input matches `fullName` after trim, case-insensitive (RESEARCH A3)
- On confirm: DELETE (soft-archive) API call → toast "Student archived" → redirect to `/students`

**States:**

| State | Visual |
|-------|--------|
| Loading | Skeleton for name, subtitle, profile fields |
| Not found (404) | Alert + "Back to students" link |
| Edit mode | Form fields editable; "Save changes" and "Discard changes" visible |
| Read mode | Field values static, Edit button visible |
| Archived (admin view) | "(Archived)" in subtitle; Archive hidden; Restore visible |

## Accessibility Baseline

Inherit WCAG 2.1 AA from 01-UI-SPEC.md. Phase 2 additions:

| Requirement | Implementation |
|-------------|---------------|
| Sortable columns | `<button>` in header with `aria-sort="ascending|descending|none"` |
| Table semantics | Native `<table>`, `<th scope="col">`, caption optional (page heading suffices) |
| Search | Input has `<label className="sr-only">Search by name</label>` or visible label |
| Filters | Each Select has visible Label element |
| Archive dialog | Focus trap (AlertDialog default); confirm disabled state announced via button `aria-disabled` |
| Row navigation | Row clickable area has clear focus style; prefer name link for keyboard users |
| Toast announcements | Sonner `role="status"` default |
| Loading | `aria-busy="true"` on table container during fetch |
| Admin toggle | Checkbox with `<label htmlFor>` |
