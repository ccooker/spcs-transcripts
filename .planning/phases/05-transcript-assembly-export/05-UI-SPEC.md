**Accent (`--primary`) reserved for Phase 5:**

1. "Export PDF" primary button in transcript editor page header
2. "Save settings" submit button on Settings page
3. Focus rings on all interactive elements (inherited)

**Accent is NOT used for:** TipTap toolbar buttons (ghost), section Show/Hide switches (default Radix Switch uses accent for `checked` state — acceptable since it represents a meaningful active state), records-updated banner, status select, the "Regenerate draft" button (use `variant="outline"`).

### Transcript Status (Phase 5 Live Data)

Phase 2 defined the badge variants; Phase 5 makes them live with real status data from the `Transcript` model. No variant changes — same mapping:

| Status | Badge Variant | Select Option Label |
|--------|--------------|---------------------|
| `NONE` | `secondary` | "None" |
| `DRAFT` | `outline` | "Draft" |
| `FINALISED` | `default` | "Finalised" |

The transcript status `Select` in the editor header shows the current status and allows changing to Draft or Finalised. Option "None" is shown for display only (cannot be manually set to None via the editor — that state is only cleared server-side if the transcript record is deleted, which is out of scope for v1).

### Hidden Section Visual Treatment

When a transcript section's show/hide Switch is toggled **off** (hidden from PDF):
- Section card header: `bg-muted/50` (stronger muted than default header)
- Section card border: `border-dashed` — signals non-active state
- Section title: `text-muted-foreground` (instead of default foreground)
- TipTap editor area: `opacity-60` — content still visible and editable; just de-emphasized
- No overlay or pointer-events block — staff can still read and edit hidden sections

When switch is **on** (included in PDF): full opacity, default border, normal foreground.

| Element | Component | Spec |

**Section order (D-06):** Academics → Activities → Awards → Work Experience → Career Goals → Staff Endorsement

**Auto-save:** 1500ms debounced after any TipTap `onUpdate` event. Each section saves independently via `PATCH /api/students/:id/transcript` with the updated section field. The page-level auto-save indicator reflects the most recent save operation across all sections.

**First open (auto-population, D-08):** When `GET /api/students/:id/transcript` returns `null` or empty narrative fields, the client renders auto-generated structured prose in each TipTap editor (except Staff Endorsement, which starts blank). This rendered content is NOT yet persisted — it is persisted only on first auto-save trigger (first edit or after 1500ms idle). Staff Endorsement section shows empty editor with placeholder copy.

**Auto-save behaviour (Claude's discretion):**

- 1500ms debounced after each TipTap `onUpdate` event per section
- Auto-save indicator in transcript page header (not per-section)
- Indicator states: "All changes saved" (default, muted) → "Saving…" (during in-flight PATCH) → "Save failed — retry?" (on error, `text-destructive`, click retries)
- Clicking "Save failed — retry?" triggers immediate retry of the failed PATCH

**PDF export trigger (Claude's discretion):**

- No preview before download — triggers `POST /api/students/:id/transcript/export` immediately on button click (per CONTEXT.md deferred: "PDF preview panel — v1 triggers download immediately")
- Response is a blob with `Content-Disposition: attachment; filename="transcript-{student-name}.pdf"`
- `fetch` → `response.blob()` → `URL.createObjectURL()` → `<a download>` programmatic click (same `fetch+blob+anchor` pattern as Phase 4 document download)

## Accessibility Baseline

Inherit WCAG 2.1 AA from 01-UI-SPEC.md, Phase 2, Phase 3, and Phase 4 additions. Phase 5 additions:

| Requirement | Implementation |
|-------------|---------------|
| Transcript page heading | `<h1>` for page title; `<h2>` for each section title inside `TranscriptSectionCard` |
| TipTap editor role | `<div role="textbox" aria-multiline="true" aria-label="{section title} narrative">` on `EditorContent` wrapper |
| TipTap toolbar | `<div role="toolbar" aria-label="{section title} text formatting">` wrapping the four toolbar buttons |
| Toolbar active state | `aria-pressed="true"` on active formatting buttons (Bold/Italic/List); `aria-pressed="false"` otherwise |
| Show/Hide switch | `<Switch id="section-{id}-visibility" />` + `<Label htmlFor="section-{id}-visibility">Include in transcript</Label>` |
| Switch state announcement | Radix Switch sets `aria-checked` natively; screen reader announces "Include in transcript, checked/unchecked" |
| Records-updated banner | `<Alert role="status" aria-live="polite">` — announced when it appears |
| Regenerate dialog | Focus trap via Radix AlertDialog; Enter triggers focused action; Escape closes |
| Status select | `aria-label="Transcript status"` on `Select` trigger (no visible label in header row) |
| Export button disabled | `disabled` attribute + `aria-disabled="true"` while exporting |
| Auto-save indicator | `aria-live="polite"` region wrapping the indicator span — announces "Saving…" and "All changes saved" transitions |
| Settings form | All `Input`/`Textarea` fields have `<Label htmlFor>` association |
| Logo file input | `<label htmlFor="logo-file-input">School logo</label>` + `aria-required="false"` on native input |
| AppShell nav — Settings | `aria-current="page"` on active nav link; `aria-label="Admin settings"` on the link |
| Letterhead textarea | `aria-describedby` linking to the hint text span below the field |
| Hidden section editor | TipTap editor remains keyboard-accessible even when hidden (`opacity-60` does not disable focus) |
| Toast announcements | Sonner `role="status"` default (inherited) |
